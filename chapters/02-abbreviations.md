::: {custom-style="Chapter Title"}
Abbreviations and terms
:::

| Abbreviation and Terms                   | Note |
|--|-------|
| AI                                       | Artificial Intelligence                                                                                                                          |
| API                                      | Application Programming Interface                                                                                                                |
| Application Programming Interface        | a set of rules and protocols for building and interacting with software applications                                                             |
| Big Picture Event Storming               | a workshop-based method to explore complex business domains and identify key events and processes                                                |
| Blueprint                                | here short for Synergetic Blueprint                                                                                                              |
| Bounded Context                          | a central pattern in DDD that defines a boundary within which a particular domain model is defined and applicable                                |
| Brainstorming                            | a group creativity technique designed to generate a large number of ideas for the solution to a problem                                          |
| Brainwriting                             | Brainwriting  a group creativity technique where participants write down their ideas on paper                                                    |
| Business Model Canvas                    | a strategic management template for developing new or documenting existing business models                                                       |
| Capability                               | a high-level description of what an organization is capable of                                                                                   |
| Capability Map                           | a visual representation of the capabilities of an organization and their relationships                                                           |
| Choreography                             | a style of microservice architecture where services communicate by producing and consuming events rather than direct calls                       |
| Domain Storytelling                      | a technique for gathering and structuring business requirements by telling stories about the domain                                              |
| Command Query Responsibility Segregation | a pattern that separates the read and write operations of a system into different models, allowing for better scalability and maintainability |
| CQRS                                     | Command Query Responsibility Segregation                                              |
| DDD                                      | Domain-Driven Design                                                                                                                             |
| Event-driven architecture                | a software architecture pattern where the flow of the program is determined by events, which can be produced and consumed by different components       |
| Event Modeling                           | a method for modeling complex business processes and systems using events and commands                                                           |
| Event Sourcing                           | a pattern where state changes are stored as a sequence of events, allowing for better traceability and the ability to reconstruct past states            |
| Event Storming                           | a workshop-based method to explore complex business domains                                                                                      |
| Impact Mapping                           | a strategic planning technique that helps teams visualize the relationships between business goals, user needs, and software features            |
| Lean Canvas                              | a one-page business plan template that helps entrepreneurs deconstruct their idea into key assumptions                                           |
| Microservice                             | a software architectural style that structures an application as a collection of loosely coupled services, which implement business capabilities |
| North Star Metric                        | a single measure of success that aligns teams around a common goal                                                                               |
| Modular monolith                         | a software architecture style that structures an application as a single unit but with well-defined modules and boundaries                            |
| Orchestration                            | a style of microservice architecture where services communicate through direct calls and a central coordinator manages their interactions        |
| Solution architecture                    | the high-level structure of a software system, including its components and their interactions, designed to meet specific business needs and goals     |
| Solution Space                           | the space of possible software solutions that can be implemented to meet the business needs and goals                                                   |
| Strategic Design                         | a part of DDD that focuses on the high-level structure of the domain and the relationships between different bounded contexts                            |
| Synergetic Blueprint                     | the structured process from business intent to running software that provides the context for effective human-AI collaboration                   |
| Tactical Design                          | a part of DDD that focuses on the design of the domain model and the implementation of the software within a bounded context                            |
| Ubiquitous Language                      | the terms and concepts that are used to describe the domain and the software                                                                     |
| Visual Glossary                          | a visual representation of the ubiquitous language and the relationships                                                                         |
| Wardley Map                              | a visual representation of the value and maturity of functions and implementing software                                                         |
